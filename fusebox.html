Si está entre los que consideran que Webpack es demasiado complejo, puede tener en cuenta FuseBox. FuseBox es un proyecto joven (pero maduro) con algunos principios claros:

  • Velocidad: la creación de una aplicación debe ser lo más rápida posible.
  • Sencillez: la configuración de un sistema de compilación no debería causar dolores de cabeza.
  • Extensibilidad: todo lo que el núcleo de FuseBox no se hace cargo, debe hacerse a través de complementos.
FuseBox se adhiere a estos principios mediante la adopción de algunos enfoques, como el uso del compilador TypeScript de forma predeterminada, la explotación de un sistema de caché potente, que permite la división de código de configuración cero, proporciona una sintaxis de configuración fácil de entender, admite un corredor de tareas integrado y proporciona conjunto de complementos que pueden cubrir todo lo que la mayoría de las aplicaciones necesitan, y muchas otras cosas
